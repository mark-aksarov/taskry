# Projects

A project is an entity that stores the main information about work and connects related data:

- Main information — <span className="font-normal">title, description, deadline</span>
- Tasks — <span className="font-normal">work items within the project</span>
- Comments — <span className="font-normal">discussions linked to the project</span>
- Customer — <span className="font-normal">information about the customer for whom the project is done</span>
- Category — <span className="font-normal">grouping of projects by type or area</span>
- Status — <span className="font-normal">the current state of the project</span>
- Created by — <span className="font-normal">the user who created the project</span>

---

## Overview

This section explains how to work with projects and the main features available to users in the system.

### Viewing projects

This section explains how projects are displayed in the system interface. It covers two view modes — list and cards — and describes what basic information users can see about a project.

### Project actions

This section covers the main project actions: creating new projects, editing projects, changing project status, and deleting projects.

### Search and filtering

This section explains how users can find the projects they need among many records, as well as use filters and sorting to narrow down the results.

### Project categories

This section covers project categories and the actions available to users — viewing, creating, editing, and deleting categories. Categories help organize projects and make it easier to find the right projects in the system.
