# Tasks

A task is a unit of work that contains the information needed to complete a specific action:

- Title — <span className="font-normal">a short name of the task</span>
- Description — <span className="font-normal">detailed information about the task</span>
- Deadline — <span className="font-normal">the date by which the task should be completed</span>
- Project — <span className="font-normal">the project the task belongs to</span>
- Assignee — <span className="font-normal">the user responsible for completing the task</span>
- Author — <span className="font-normal">the user who created the task</span>
- Category — <span className="font-normal">a logical group for the task</span>
- Status — <span className="font-normal">the current state of the task</span>
- Subtasks — <span className="font-normal">smaller parts of the task</span>
- Comments — <span className="font-normal">discussions and notes related to the task</span>

## Overview

This section explains how to work with tasks and the main features available to users in the system.

### Viewing tasks

This page explains how tasks are displayed in the system interface. It covers two view modes — list and cards — and shows what basic information users can see about a task.

### Task actions

This page covers the main task actions: creating tasks, editing tasks, changing task status, and deleting tasks.

### Search and filtering

This page explains how users can find tasks among many records. It also covers filtering and sorting that help narrow down the results.

### Subtasks

This page covers the main subtask actions: creating subtasks, editing subtasks, changing subtask status, and deleting subtasks.

### Task categories

This page explains task categories and the actions available to users: viewing categories, creating categories, editing categories, and deleting categories. Categories help organize tasks and make it easier to find the needed tasks in the system.
